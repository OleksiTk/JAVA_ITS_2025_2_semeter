#### 1. Призначення JFC фреймворку і його архітектура [cite: 23]

**JFC (Java Foundation Classes)** — це набір бібліотек Java для розроблення графічних інтерфейсів (GUI) та додавання багатої графічної функціональності й інтерактивності клієнтським програмам.

**Архітектура JFC включає:**

- **AWT (Abstract Window Toolkit):** Базовий набір компонентів, що використовує елементи ОС.
- **Swing:** Бібліотека нових, легковагових компонентів, побудована поверх AWT (наприклад, JButton, JFrame).
- **Java 2D:** Для малювання 2D-графіки.
- **Accessibility API:** Для забезпечення доступу людям з обмеженими можливостями (читання з екрана тощо).
- **Drag and Drop:** Підтримка перетягування об'єктів.

#### 2. Які переваги графічної бібліотеки Swing у порівнянні з бібліотекою AWT [cite: 24]

Головна відмінність: компоненти AWT є "великоваговими" (Heavyweight), а Swing — "легковаговими" (Lightweight).

**Переваги Swing:**

1.  **Платформонезалежний вигляд:** Swing малює компоненти самостійно засобами Java, тому вони виглядають однаково на Windows, Linux та MacOS. AWT використовує готові кнопки операційної системи, тому вигляд залежить від ОС.
2.  **Більше компонентів:** Swing має складні таблиці (`JTable`), дерева (`JTree`), вкладки (`JTabbedPane`), яких немає в AWT.
3.  **Гнучкість (Pluggable Look-and-Feel):** Можна змінювати тему оформлення програми "на льоту".
4.  **MVC Архітектура:** Компоненти Swing побудовані за принципом Model-View-Controller, що полегшує роботу з даними.

#### 3. Навести приклад структури GUI програми на Java і пояснити призначення її елементів [cite: 25]

Типова структура (як у наданому коді):

1.  **Imports:** `import javax.swing.*;` — підключення бібліотек.
2.  **Class Definition:** `public class MyApp extends JFrame` — головний клас успадковує вікно.
3.  **Constructor:** Метод `MyApp()`, де налаштовується:
    - `Layout Manager` (менеджер розташування).
    - Створення компонентів (кнопки, поля).
    - Додавання компонентів у контейнер (`add()`).
    - Реєстрація слухачів подій (`ActionListener`).
4.  **Main Method:** `public static void main...` — точка входу, яка запускає вікно в окремому потоці (`SwingUtilities.invokeLater`).

#### 4. Пояснити призначення Containers в Awt/Swing. Що може використовуватися в якості Containers? [cite: 26]

**Container (Контейнер)** — це компонент, який може містити в собі інші компоненти (кнопки, текстові поля) або інші контейнери. Він відповідає за розміщення елементів усередині себе.

**Типи контейнерів:**

- **Top-Level Containers (Верхнього рівня):** Це основа вікна.
  - `JFrame` (головне вікно з рамкою).
  - `JDialog` (діалогове вікно).
  - `JApplet` (для браузера, застаріле).
- **Intermediate Containers (Проміжні):** Використовуються для групування елементів всередині вікна.
  - `JPanel` (панель).
  - `JScrollPane` (панель з прокруткою).
  - `JTabbedPane` (вкладки).

#### 5. Навести приклади основних методів із класу Frame, що використовуються для розроблення GUI Java додатків [cite: 27]

Клас `JFrame` (або `Frame` в AWT) має методи для керування вікном:

- `setSize(width, height)` — встановити розмір вікна.
- `setVisible(true)` — показати вікно на екрані.
- `setTitle("Title")` — встановити заголовок.
- `add(component)` — додати елемент у вікно.
- `setLayout(manager)` — встановити схему розташування елементів.
- `setDefaultCloseOperation(int operation)` — визначити дію при натисканні на "хрестик" (наприклад, вихід з програми).
- `setLocationRelativeTo(null)` — центрувати вікно.

#### 6. Пояснити призначення об'єкта Panel. Як додати Components до Panel? [cite: 28]

**Panel (`JPanel`)** — це контейнер загального призначення. Він не має власного вікна (рамок, кнопок закриття), а служить для логічного групування кнопок чи полів, щоб потім розмістити цю групу у головному вікні. Це допомагає створювати складні інтерфейси.

**Як додати компоненти:**

```java
JPanel panel = new JPanel(); // Створення панелі
JButton btn = new JButton("OK"); // Створення кнопки
panel.add(btn); // Додавання кнопки на панель
frame.add(panel); // Додавання панелі на головне вікно
```
